# Контрольная работа №3 — Приложение «Заметки» 

## Общее описание

В рамках практических занятий №13–18 разработано веб-приложение для управления заметками. Приложение работает офлайн, поддерживает push-уведомления, WebSocket‑обновления в реальном времени, напоминания и безопасное HTTPS‑соединение.

---

## Практическое занятие №13 — Service Worker

- Регистрация Service Worker.
- Кэширование статических ресурсов (HTML, CSS, JS).
- Стратегия `Cache First` для всех запросов в `fetch`.
- Хранение заметок в `localStorage` с возможностью добавления и просмотра без интернета.

---

## Практическое занятие №14 — Web App Manifest

- Создан `manifest.json` с полями: `name`, `short_name`, `start_url`, `display`, `background_color`, `theme_color`, `description`, `orientation`, `icons`.
- Подготовлены иконки разных размеров в папке `/icons`.
- В `index.html` добавлены: `<link rel="manifest">`, мета‑теги для iOS и Android, `theme-color`.
- Иконки и манифест добавлены в список кэширования Service Worker.

---

## Практическое занятие №15 — HTTPS + App Shell

- Сгенерированы самоподписанные сертификаты через `mkcert` для `localhost`.
- Сервер запущен по HTTPS.
- Внедрена архитектура App Shell:
  - `index.html` содержит только шапку, навигацию и контейнер `<main>`.
  - Контент страниц (`home.html`, `about.html`) подгружается динамически через `fetch`.
  - Service Worker кэширует App Shell (Cache First), а динамические страницы загружаются по стратегии Network First с fallback на `home.html`.
- Проверена работа в офлайн‑режиме: каркас загружается мгновенно.

---

## Практическое занятие №16 — WebSocket + Push

- Сервер на Node.js + Express + Socket.IO + `web-push`.
- Сгенерированы VAPID‑ключи.
- На клиенте:
  - Подключение к серверу через Socket.IO.
  - Отправка события `newTask` при добавлении заметки.
  - Получение события `taskAdded` — показ временного всплывающего сообщения.
  - Подписка/отписка на push‑уведомления через `PushManager`.
  - Кнопки «Включить уведомления» / «Отключить уведомления».
- Service Worker обрабатывает событие `push` и показывает системное уведомление.
- Сервер хранит подписки в массиве и рассылает push при получении `newTask`.

---

## Практическое занятие №17 — Детализация Push 

- В `home.html` добавлена форма с полями: текст напоминания и `datetime-local`.
- В `app.js`:
  - Заметки хранятся с полями `id`, `text`, `reminder` (timestamp), `reminderText`.
  - При отправке формы с напоминанием данные отправляются на сервер через событие `newReminder`.
- Сервер (`server.js`):
  - Хранилище активных таймеров `reminders` (Map).
  - При получении `newReminder` вычисляется задержка, создаётся `setTimeout`.
  - По истечении таймера отправляется push‑уведомление всем подписанным клиентам.
  - Добавлен эндпоинт `POST /snooze` — откладывает напоминание на 5 минут (сброс старого таймера + новый).
- Service Worker:
  - В уведомлении появляется кнопка «Отложить на 5 минут» (`actions`).
  - Обработчик `notificationclick` при нажатии на кнопку отправляет `fetch` на `/snooze`.
